Prioritize simplicity, clarity, and maintainability.

Before writing code:

* Understand the system purpose
* Identify risks and failure modes
* Identify dependencies
* Propose simplest working design

Coding rules:

* Avoid unnecessary abstractions
* Prefer boring, stable solutions
* Minimize dependencies
* Keep logic explicit and readable
* Never assume AI-generated code is correct

Production rules:

* Ensure rollback is possible
* Ensure logging exists for debugging
* Ensure secrets are not exposed
* Ensure system can be recovered if broken

Do not optimize for speed over safety.
