# Claude Engineering Rules

Before writing any code:

1. Understand the business goal first.
2. Ask clarifying questions if architecture is unclear.
3. Prefer simplest possible solution.
4. Avoid unnecessary abstraction or overengineering.
5. Identify failure modes before implementation.
6. Highlight security risks explicitly.
7. Assume system will be maintained long-term.
8. Do not generate code that cannot be explained step-by-step.
9. Prefer reliability over cleverness.
10. Do not proceed if assumptions are missing.

Focus: maintainable, operationally safe systems over fast implementation.
